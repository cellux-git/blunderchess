# 10 — King opposition in endgames

**Status:** `completed`

**Status**: `completed`
**Category**: `enhancement`

## What's missing

In king-and-pawn endgames, king opposition (the two kings facing each other on the same file or rank with exactly one square between them) is often decisive. The engine has no opposition detection.

## What to implement

A general king opposition bonus, gated by two conditions:

### Detection

Kings are on the same file or same rank, exactly 2 squares apart (distance = 2 Manhattan, same file OR same rank), with nothing between them.

```rust
pub king_opposition_bonus: i32, // flat bonus, e.g. +50
```

### Gate 1: No non-pawn, non-king enemy pieces

If the opponent has any queens, rooks, bishops, or knights remaining, skip the bonus. Opposition only matters when the board is stripped down to kings and pawns.

### Gate 2: Opponent cannot waste a tempo

If the opponent has any safe move (any legal move that doesn't lose material per SEE ≥ 0), skip the bonus. The opposition only benefits the side that already has it — if the opponent can make a harmless waiting move, the opposition flips next turn.

### Application

Bonus is awarded to the side that currently has the opposition (kings are facing each other distance-2, same file or rank). This is a static eval term — the bonus reflects that the side WITH the opposition has a positional advantage.

## Key interfaces

- `Eval` struct — add `king_opposition_bonus: i32`
- New `eval_king_opposition` method — check opposition pattern, apply gates, award bonus
- `evaluate_side` — call the new method

## Acceptance criteria

- [ ] King opposition detected (kings on same file/rank, distance 2, nothing between)
- [ ] Bonus skipped when opponent has Q/R/B/N
- [ ] Bonus skipped when opponent has any safe tempo-wasting move
- [ ] K+P vs K with opposition scores higher than without
- [ ] All existing tests pass
- [ ] No NPS regression

## Out of scope

- K+PP vs K+P opposition evaluation
- Distant opposition (diagonal, horizontal distance > 2)
- Mined squares / corresponding squares
