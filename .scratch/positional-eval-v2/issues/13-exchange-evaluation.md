# 13 — Exchange evaluation

**Status**: `completed`
**Category**: `enhancement`

## What's missing

"The exchange" — trading a rook for a bishop or knight (material difference ~170cp) — is evaluated as pure material. Positional context determines whether it's a good or bad exchange:
- Open files favor the rook side
- Opponent keeping the bishop pair penalizes the exchange
- Minor piece activity (is the remaining minor a strong outpost knight or a bad bishop?)

## What to implement

Detect when one side is up the exchange (has rook, opponent has bishop/knight instead). Apply positional adjustments from 3 independent factors, each with their own Eval field:

### 1. Open files (bonus for rook side)

Per open or semi-open file the rook side controls, add a bonus. The rook thrives on open lines.

```rust
pub exchange_open_file_bonus: (i32, i32), // mg, eg — per open/semi-open file
```

### 2. Bishop pair (penalty for rook side)

If the opponent retains the bishop pair after the exchange, the rook side is penalized. Bishop pair is worth more than the raw material difference suggests.

```rust
pub exchange_bishop_pair_penalty: (i32, i32), // mg, eg
```

### 3. Minor piece activity

If the opponent's remaining minor piece is strong (high mobility, outpost), the exchange is worse. If it's passive (bad bishop, low mobility, no safe squares), the exchange is better.

```rust
pub exchange_minor_activity_bonus: (i32, i32), // mg, eg — bonus when opponent's minor is passive
```

Use the minor piece's mobility count relative to its max: if ≤ 25% of max mobility, apply bonus. If ≥ 75% of max mobility, apply penalty.

### Application

When the imbalance is exactly one side up a rook and the other side up a minor piece (knight or bishop), sum the 3 factors. The adjustment is symmetric — positive from the rook side's perspective.

## Key interfaces

- `Eval` struct — add 3 new field pairs (6 fields total)
- New `eval_exchange` method — detect imbalance, apply factors
- `evaluate_side` — call the new method
- `Eval::default()` — sensible defaults (e.g. open file: +10/+15, bishop pair: -20/-30, passive minor: +15/+20)

## Acceptance criteria

- [ ] Exchange imbalance (rook vs minor) detected and positional adjustments applied
- [ ] Rook side with open files scores higher than without
- [ ] Rook side penalized more when opponent has bishop pair
- [ ] Rook side benefits when opponent's minor piece is passive (bad bishop, trapped knight)
- [ ] Neutral exchange (no open files, no bishop pair, average minor) ≈ pure material
- [ ] All existing tests pass
- [ ] No NPS regression

## Out of scope

- Double exchange (rook+minor vs 2 minors)
- Queen-on-board exchange dynamics (handled implicitly by normal eval)
- Rook placement / king safety adjustments (already covered by existing eval terms)
